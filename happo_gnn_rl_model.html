物理先验 式(3.1)
Sij = MI(i, j) + Granger(i → j) S_ij：站点i与站点j的关联强度  |  MI：互信息  |  Granger：格兰杰因果贡献度

通俗讲解

先把站点之间的"亲疏远近"关系算清楚。互信息MI衡量两个站点充电需求在统计上的相关程度（懂的自然懂）；格兰杰因果检验则判断"站点i的充电行为能不能预测站点j的"，如果能，说明i对j有因果影响力。把这两项加在一起，就得到站点对(i,j)的物理关联强度S_ij——值越大，两个站点联系越紧密，在后面的注意力机制里就越容易互相"注意到"对方。

注意力分数 式(3.2)
ωij = LeakyReLU( Wa · [hi ⊕ hj ] ) ω_ij：站点对(i,j)的注意力分数  |  W_a：可学习权重矩阵  |  ⊕：拼接操作  |  h：节点特征

通俗讲解

这一步用神经网络自己学出来的注意力分数。两个站点的特征向量h_i和h_j拼在一起[ ]，扔进一个带LeakyReLU激活的全连接层，出来就是一个数字ω_ij。这个数字代表"站点i在决定自己行为的时候，应该给站点j多少关注"。神经网络会自己调整W_a的参数，让这个分数在训练中越来越合理。

注意力融合 式(3.3)
eij = ωij + γ · Sij e_ij：融合后的注意力分数  |  ω_ij：神经网络学出来的注意力  |  γ：物理先验权重  |  S_ij：物理先验关联

通俗讲解

把神经网络自己学出来的注意力ω_ij，和物理领域知识S_ij结合起来。γ是物理先验的权重（超参数），γ越大，说明越相信物理规律；γ越小，说明越让神经网络自己决定谁关注谁。这一步既利用了专家知识（物理规律），又保留了数据驱动（神经网络学出来的关系），两者互补，缺一不可。

Softmax归一化 式(3.4)
αij = softmax( eij ) = exp(eij) / Σk exp(ekj) α_ij：归一化注意力权重（所有站点对同一j的权重之和为1）

通俗讲解

Softmax把所有融合注意力e_ij变成概率分布。对于"节点j关注哪些邻居"这件事，把所有e_kj取指数exp，再除以总和，归一化到[0,1]区间，且所有邻居的α加起来等于1。这么做的目的是：让每个节点j的注意力权重是一个合法的概率分布，方便后面加权聚合，也保证了数值稳定性。

特征聚合 式(3.5)
h'i = Σj∈N(i) αij · WV · hj h'：聚合后的节点特征  |  α_ij：注意力权重  |  W_V：值投影矩阵  |  N(i)：节点i的邻居集合

通俗讲解

加权平均大法好！每个节点i的新特征h'_i，等于所有邻居节点j的特征h_j（先经过W_V投影），乘以注意力权重α_ij，再求和。用通俗的话说就是："我h'_i参考了周围邻居们的意见，参考多少取决于他们和我的关系有多铁（α_ij）"。图神经网络的精髓就在这里——每个节点不只是自己的特征，还汇聚了全图的结构信息。

多头注意力 式(3.6)
hi(l+1) = ⊕k=1K σ( Wk(l) · hi(l) + bk(l) ) ⊕：多向量拼接  |  K：头数  |  σ：激活函数  |  l：网络层索引

通俗讲解

做K次独立的注意力计算（K个头），每次学一套不同的W_k,b_k，捕获不同类型的关联模式（比如时间关联、空间关联、需求关联等）。最后把K个结果直接拼起来（⊕），得到更丰富、更全面的特征表达。这就像让K个专家各自分析一个问题，最后综合所有人的意见做决策，比一个人强得多。

残差连接 式(3.7)
hi(l+1) = σ( h'i ) + hi(l) σ：激活函数  |  h'_i：GAT聚合结果  |  h_i^(l)：上一层输入（恒等映射）

通俗讲解

把GAT算出来的新特征h'_i（经过激活σ），直接加到上一层的输入h_i^l上。这就是ResNet的核心思想——不要求网络每次都重新学完整的变换，只要求它学"要改变多少"（残差）。这样即使堆很多层网络，梯度也能顺畅地传回去，不会因为层数太多而"信息衰减"，训练更稳定，特征提取更有效。

策略网络 式(3.8)
z1 = W1 · hi + b1
z3 = W2 · LeakyReLU(z1) + b2 z_1：第一层隐状态  |  z_3：第二层输出（未归一化动作概率） |  W：权重矩阵

通俗讲解

策略网络就是一个双层MLP。第一层：输入GAT提取出来的站点特征h_i，线性变换得到z_1；第二层：对z_1做LeakyReLU激活（让负值也能有一点梯度），再线性变换得到z_3，z_3的维度等于该车型可执行的动作数，数值上表示"每个动作的得分"（未归一化），得分越高越可能被选择。这就是一个"输入状态，输出动作偏好"的神经网络。

动作掩码 式(3.9)
πmasked(ai | si) = πθ(ai | si) ⊙ M(si) π_θ：原始策略网络输出  |  M(s_i)：动作掩码向量（0/1） |  ⊙：逐元素乘法

通俗讲解

直接用网络输出的动作概率有个问题——不是所有动作在当前状态下都合法。比如电量为100%时"充电"这个动作就没意义。掩码M(s_i)就是一个0/1向量：非法动作位置填0，合法动作位置填1。用⊙逐元素乘，就把非法动作的概率直接变成0，合法动作概率保持不变——但这还没完，概率和不等于1了，需要下一步重归一化。

重归一化 式(3.10)
πnorm(ai | si) = πmasked(ai | si) / Σ πmasked π_norm：合法概率之和等于1  |  非法动作概率→0  |  合法动作概率按比例重新分配

通俗讲解

掩码把非法概率清零后，总概率和小于1了，不是一个合法的概率分布。怎么办？除以总和。这样所有合法动作的概率按比例放大，重新变成一个合法的概率分布（总和=1）。从数学上保证了无论掩码怎么变，π_norm永远是一个合法的概率分布，策略网络可以稳定训练。

动作选择 式(3.11)
ai* = arg maxai πnorm(ai | si) a_i*：最终选择的充电动作  |  argmax：选择概率最大的合法动作

通俗讲解

贪心策略：在所有合法动作中，挑选概率π_norm最大的那个，作为最终执行的充电动作a_i*。这就是推理阶段（inference）的决策方式——"哪个动作概率最高就选哪个"。训练时用随机采样（探索），推理时用argmax（利用），这是强化学习的经典套路。

奖励函数 式(3.12)
Rt = −( w1 · Tt + w2 · Pt + w3 · Lt − w4 · Gt ) T_t：等待时间  |  P_t：充电费用  |  L_t：充电损失  |  G_t：电网收益  |  w_i：权重系数  |  负号→最小化=最大化奖励

通俗讲解

强化学习的核心——给每个动作打分。打分原则：等待时间T越长、电费P越高、充电损失L越大，奖励R越低（被惩罚）；电网收益G越高，奖励R越高（被奖励）。括号前面加负号，说明这是一个"最小化问题变最大化"的经典技巧——让等待时间等成本越小越好，电网收益越大越好。所有w_i都是权重，可调节各因素的相对重要性。

PPO裁剪损失 式(3.13)
LCLIP = Et[ min( rt · At, clip(rt, 1−ε, 1+ε) · At ) ] r_t = π_θ/π_θ_old（概率比） |  A_t：优势函数  |  ε：裁剪区间（通常0.2） |  E：期望

通俗讲解

这是HAPPO的精髓。r_t是新旧策略概率的比值，衡量"这个动作现在比过去好多少"。clip把r_t夹在[1-ε, 1+ε]区间——如果新策略提升太多（r_t > 1+ε），clip把增长砍掉；如果新策略大幅退步（r_t < 1-ε），同样不允许变差超过限制。这样就防止策略在一次更新中变动太大而崩溃。每次更新后θ_old更新为θ_new，重复迭代，策略逐步变强。

梯度更新 式(3.14)
θnew = θold − η · ∇θ LCLIP η：学习率（步长） |  ∇：梯度符号  |  θ：策略网络所有参数  |  负号：梯度下降方向

通俗讲解

神经网络的"学习"过程：对损失L_CLIP求关于θ的梯度∇_θ，再沿着梯度的反方向（因为要最小化损失）走一小步η（学习率）。就这么简单粗暴：梯度指向损失增加最快的方向，反方向就是损失减小最快的方向。每次更新一点点，重复成千上万次，策略网络就越来越会做充电调度决策了。

算法 3.1  基于物理先验的GAT空间特征提取
Algorithm 3.1
# 输入：站点特征 H^(0)，物理先验矩阵 S，层数 L，头数 K
# 输出：空间增强特征 H^(L)

Step 1.  构建物理先验关联矩阵
    对所有站点对 (i, j)，计算：
    S_ij ← MI(i, j) + Granger(i → j)
    构建邻接矩阵 A，将 S_ij 作为边权重

Step 2.  多层 MH-Res-GAT 前向传播
    for l = 1 to L do:
    # 多头注意力计算
    for k = 1 to K do:
    ω_ij ← LeakyReLU(W_a · [h_i ⊕ h_j])
    e_ij ← ω_ij + γ · S_ij           # 融合物理先验
    α_ij ← softmax_j(e_ij)           # 归一化权重
    h'_i ← Σ_j α_ij · W_V · h_j      # 加权聚合
    h_i^mid ← CONCAT( h'_i for k = 1..K )  # K头拼接
    h_i^(l) ← σ(h_i^mid) + h_i^(l-1)  # 残差连接
    end for

Step 3.  输出空间增强特征
    return H^(L) ← [ h_1^(L), h_2^(L), ..., h_N^(L) ]

算法 3.2  掩码策略网络与动作选择
Algorithm 3.2
# 输入：站点特征 h_i（来自GAT），当前状态 s_i，策略网络参数 θ
# 输出：最优充电动作 a_i*

Step 1.  策略网络前向传播
    z_1 ← W_1 · h_i + b_1                # 第一层
    z_3 ← W_2 · LeakyReLU(z_1) + b_2    # 第二层（未归一化动作分数）
    π_θ ← softmax(z_3)              # 原始动作概率分布

Step 2.  生成并应用动作掩码
    M(s_i) ← get_mask(s_i)         # 根据状态生成掩码（0=非法，1=合法）
    π_masked ← π_θ ⊙ M(s_i)             # 逐元素乘法，非法动作概率→0

Step 3.  重归一化
    π_norm ← π_masked / Σ π_masked     # 重新归一化为合法概率分布

Step 4.  推理：贪心动作选择
    a_i* ← argmax_a_i π_norm(a_i | s_i) # 选择概率最高的合法动作
    return a_i*

# 训练时：π_norm 用于采样与环境交互
# 推理时：argmax 用于贪心决策

算法 3.3  HAPPO-GNN-RL 完整训练主循环
Algorithm 3.3
# HAPPO-GNN-RL 充电站调度多智能体强化学习训练框架
# 输入：站点特征 H，物理先验 S，最大迭代 T_max，批次 N

Phase 1. GAT 空间特征提取（所有站点并行）
    H ← MH-Res-GAT( H^(0), S )      # 算法3.1，获得空间增强特征

Phase 2. 多智能体轨迹收集
    for episode = 1 to N do:
    for 每个站点 i in 充电站集合:
    a_i ← π_norm(h_i)             # 算法3.2：掩码→归一化→采样
    执行动作 a_i，获得奖励 r_t
    观测下一状态 s_{i,t+1}
    存储轨迹：{ s_i, a_i, r_t, s_i' }
    end for
    end for

Phase 3. GAE 优势函数估计
    for 每个智能体 i，车型优先级 order = [type_A, ..., type_D]:
    计算 n步回报：G_t ← Σ_{k=0}^{n-1} γ^k · r_{t+k}
    计算GAE优势：
    A_t^GAE ← Σ_{k=0}^{∞} (γλ)^k · δ_{t+k}
    δ_k ← r_k + γ·V(s_{k+1}) − V(s_k)

Phase 4. HAPPO 顺序策略更新
    for i in order（按车型优先级顺序）:
    # 更新智能体i的策略，其他智能体策略冻结
    r_t ← π_θ(a_i | s_i) / π_θ_old(a_i | s_i)
    L_CLIP ← min( r_t · A_t, clip(r_t, 1−ε, 1+ε) · A_t )
    θ_i ← θ_i − η · ∇_θ L_CLIP           # 梯度下降更新
    π_θ_old(i) ← π_θ(i)                 # 保存新策略
    end for

Phase 5. 评估与下一轮
    θ_old ← θ_new
    重复 Phase 1 ~ 4，直到收敛或 T ≥ T_max
